# 项目开发路线图 (Roadmap)

## 第一阶段：MVP 简易 Demo (Local Script)
**目标:** 在本地电脑上运行，通过命令行触发，完成"生成 -> 发布"的单次闭环。

*   **架构:** 单机 Python 脚本 (CLI)。
*   **核心技术栈:**
    *   **语言:** Python 3.10+
    *   **浏览器自动化:** Playwright (Python版)
    *   **AI 接口:** 调用 Gemini API (或其他) 生成简单的图文内容。
*   **功能列表:**
    1.  **手动登录:** 用户首次运行需扫码登录，脚本将 Cookie 保存到本地文件 (`user_data/`).
    2.  **简单内容生成:** 给定一个主题，生成一段文案和一张占位图/简单AI图。
    3.  **自动上传:** 脚本自动打开浏览器 -> 进入创作者中心 -> 上传图文 -> 点击发布。
*   **交付物:** `main.py` 脚本。

## 第二阶段：功能丰富与桌面化 (Enhanced Desktop App)
**目标:** 即使是非技术人员也能使用，支持简单的排程和多账号。

*   **架构:** 本地桌面应用。
*   **核心技术栈:** 
    *   **GUI:** PyQt6 或 Streamlit (更轻量)。
    *   **数据存储:** SQLite (本地数据库存储发布记录、账号信息)。
*   **功能列表:**
    1.  **多账号管理:** 支持切换不同的小红书/抖音账号 (通过切换 Cookie 文件)。
    2.  **内容编辑器:** 用户可以修改 AI 生成的内容后再发布。
    3.  **定时任务:** 简单的本地定时器 (Schedule 库)。
    4.  **多平台适配:** 抽象出 `Publisher` 基类，适配抖音 Web 端。

## 第三阶段：商业化 Web 服务 (SaaS Platform)
**目标:** 变成一个云端服务，用户无需一直开着电脑，支持多用户并发。

*   **架构:** 前后端分离，容器化部署。
*   **核心技术栈:**
    *   **前端:** React + Ant Design / Tailwind CSS。
    *   **后端:** FastAPI (Python) 或 Django。
    *   **任务队列:** Celery + Redis (处理耗时的发布任务)。
    *   **数据库:** PostgreSQL。
    *   **浏览器环境:** Docker + Headless Chrome (需解决无头浏览器的指纹被抓问题)。
*   **挑战:**
    *   **IP 代理池:** 服务器 IP 很容易被平台拉黑，必须集成代理 IP 服务。
    *   **云端 Cookie 保活:** 只有 Web 端很难维持 Cookie 有效期，可能需要用户定期回填 Cookie 或开发浏览器插件辅助获取 Cookie。
